    Technology Stack

    This page documents all technologies used in the project and why they were chosen.

    Overview

    Layer Technology Purpose
    Backend Laravel 8.x (PHP) Web framework, API proxy
    Frontend Vanilla JavaScript UI interactions
    Video Streaming LiveKit WebRTC video/audio
    AI Avatar HeyGen API Avatar generation
    Speech Web Speech API Voice recognition

    Backend Technologies

    PHP 7.3+

    Purpose: Server-side programming language

    Why PHP:

    • Laravel is built on PHP
    • Easy deployment on most hosting providers
    • Good documentation and community support

    Laravel 8.x

    Purpose: PHP web framework

    Features Used:

    Feature Usage
    Routing Define URL endpoints
    Controllers Handle business logic
    Configuration Manage environment variables
    HTTP Client Make API requests to HeyGen
    CSRF Protection Secure form submissions
    Blade Templates Render views

    Why Laravel:

    • Popular, well-documented framework
    • Built-in HTTP client for API calls
    • Easy configuration management
    • Strong security features (CSRF, etc.)

    Guzzle HTTP (via Laravel)

    Purpose: HTTP client for API requests

    Usage: 

    Http::withHeaders([
    'Authorization' => 'Bearer ' . $token,
])->post($url, $data);

    Laravel's Http facade wraps Guzzle, providing a clean API for HTTP requests.

    Frontend Technologies

    HTML5

    Purpose: Page structure

    Features Used:

    Feature Usage
    <video> Display avatar video stream
    <input> Text input fields
    <button> Interactive buttons
    Semantic elements Proper document structure

    CSS3

    Purpose: Styling and layout

    Features Used:

    Feature Usage
    CSS Variables Theme switching (dark/light mode)
    Flexbox Layout components
    Grid Page layout
    Transitions Smooth animations
    Media Queries Responsive design

    CSS Variables Example: 

    :root {
    --bg-primary: #f5f5f5;
}

[data-theme="dark"] {
    --bg-primary: #0f0f0f;
}

    Vanilla JavaScript (ES6+)

    Purpose: Frontend interactivity

    Why Vanilla JS (no framework):

    • No build step required
    • Simpler for demo purposes
    • Faster page load
    • Easier to understand

    Features Used:

    Feature Usage
    async/await Asynchronous API calls
    fetch() HTTP requests
    Event listeners User interactions
    DOM manipulation UI updates
    localStorage Theme persistence

    External Services

    HeyGen Streaming API

    Purpose: AI avatar generation and control

    Endpoints Used:

    Endpoint Purpose
    /v1/streaming.new Create session
    /v1/streaming.start Start streaming
    /v1/streaming.task Send text to avatar
    /v1/streaming.stop Stop session

    Documentation: docs.heygen.com

    LiveKit

    Purpose: Real-time video/audio streaming

    What is LiveKit: LiveKit is an open-source WebRTC platform for real-time communication. HeyGen uses LiveKit to stream the avatar video to your browser.

    How it's used:

    1. HeyGen returns LiveKit credentials (URL + access token)
    2. Frontend creates LiveKit Room
    3. Connects using provided credentials
    4. Subscribes to video/audio tracks
    5. Displays in HTML5 video element

    Library: livekit-client.umd.min.js (loaded from CDN)

    <script src="https://cdn.jsdelivr.net/npm/livekit-client/dist/livekit-client.umd.min.js"></script>

    Documentation: docs.livekit.io

    Browser APIs

    Web Speech API

    Purpose: Speech recognition (voice input)

    Browser Support:

    Browser Support
    Chrome ✅ Full
    Edge ✅ Full
    Safari ✅ Full
    Firefox ⚠️ Limited

    Usage: 

    const SpeechRecognition = window.SpeechRecognition || 
                          window.webkitSpeechRecognition;

const recognition = new SpeechRecognition();
recognition.lang = 'en-US';
recognition.start();

recognition.onresult = (event) => {
    const transcript = event.results[0][0].transcript;
};

    MediaStream API

    Purpose: Handle video/audio streams

    Usage: 

    const mediaStream = new MediaStream();
mediaStream.addTrack(videoTrack);
videoElement.srcObject = mediaStream;

    Development Tools

    Composer

    Purpose: PHP dependency management

    Commands: 

    composer install    # Install dependencies
composer update     # Update dependencies

    Artisan (Laravel CLI)

    Purpose: Laravel command-line interface

    Common Commands:

    Command Purpose
    php artisan serve Start development server
    php artisan config:clear Clear config cache
    php artisan cache:clear Clear application cache
    php artisan key:generate Generate app key

    Architecture Decision Summary

    Decision Choice Reason
    Backend Framework Laravel Popular, well-documented, good HTTP client
    Frontend Framework None (Vanilla JS) Simplicity, no build step
    Video Streaming LiveKit HeyGen's choice, we just connect
    AI Service HeyGen Required for avatar functionality
    Voice Input Web Speech API Native browser API, no dependencies
    Theming CSS Variables Simple, no JS library needed
