    Backend: AvatarController.php

    Constructor

    class AvatarController extends Controller
{
    private $apiUrl;      // HeyGen API base URL
    private $apiToken;    // Your HeyGen API key
    private $avatarId;    // Which avatar to use

    public function __construct()
    {
        // Load configuration from config/services.php (which reads from .env)
        $this->apiUrl = config('services.heygen.api_url');
        $this->apiToken = config('services.heygen.api_token');
        $this->avatarId = config('services.heygen.avatar_id');
    }
}

    Explanation: The constructor loads HeyGen configuration from Laravel's config system. This allows credentials to be stored securely in .env and accessed via config().

    createSession() Method

    public function createSession(Request $request)
{
    // Step 1: Create a new streaming session
    $response = Http::withHeaders([
        'Authorization' => 'Bearer ' . $this->apiToken,
    ])->post($this->apiUrl . '/v1/streaming.new', [
        'version' => 'v2',
        'avatar_id' => $this->avatarId,
    ]);

    $sessionInfo = $response->json();

    // Step 2: Start the streaming session
    $startResponse = Http::withHeaders([
        'Authorization' => 'Bearer ' . $this->apiToken,
    ])->post($this->apiUrl . '/v1/streaming.start', [
        'session_id' => $sessionInfo['data']['session_id'],
    ]);

    // Return session info with LiveKit credentials to frontend
    return response()->json([
        'success' => true,
        'session_info' => $sessionInfo['data']
        // Contains: session_id, url (LiveKit), access_token
    ]);
}

    Key Points:

    Line Purpose
    Http::withHeaders() Laravel HTTP client with Bearer token
    /v1/streaming.new Creates new HeyGen session
    /v1/streaming.start Starts the video stream
    $sessionInfo['data'] Contains session_id, url, access_token

    sendTask() Method

    public function sendTask(Request $request)
{
    // Validate input
    $request->validate([
        'session_id' => 'required|string',
        'text' => 'required|string',
        'task_type' => 'required|in:talk,repeat'
    ]);

    // Send text to HeyGen for avatar to speak
    $response = Http::withHeaders([
        'Authorization' => 'Bearer ' . $this->apiToken,
    ])->post($this->apiUrl . '/v1/streaming.task', [
        'session_id' => $request->session_id,
        'text' => $request->text,
        'task_type' => $request->task_type,  // 'talk' or 'repeat'
    ]);

    return response()->json(['success' => true]);
}

    Validation Rules:

    Field Rule Meaning
    session_id required\|string Must be provided, must be string
    text required\|string Must be provided, must be string
    task_type required\|in:talk,repeat Must be either "talk" or "repeat"

    Creating a Session (Frontend)

    async function createSession() {
    // 1. Call backend to create session
    const response = await fetch(`${API_BASE_URL}/avatar/session/create`, {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'X-CSRF-TOKEN': CSRF_TOKEN  // Laravel CSRF protection
        }
    });

    const data = await response.json();
    sessionInfo = data.session_info;

    // 2. Initialize LiveKit
    room = new LivekitClient.Room();

    // 3. Connect to LiveKit room using HeyGen-provided credentials
    await room.connect(sessionInfo.url, sessionInfo.access_token);

    // 4. Handle incoming video/audio tracks
    room.on(LivekitClient.RoomEvent.TrackSubscribed, (track) => {
        if (track.kind === 'video' || track.kind === 'audio') {
            mediaStream.addTrack(track.mediaStreamTrack);
            mediaElement.srcObject = mediaStream;
            mediaElement.play();
        }
    });
}

    Key Concepts:

    Concept Explanation
    X-CSRF-TOKEN Laravel security token to prevent cross-site request forgery
    LivekitClient.Room() Creates new LiveKit room instance
    room.connect() Connects to LiveKit server using token from HeyGen
    TrackSubscribed Event fired when video/audio track is available

    Speech Recognition Setup

    function initializeSpeechRecognition() {
    // Check for browser support (Chrome/Edge prefix)
    const SpeechRecognition = window.SpeechRecognition || 
                              window.webkitSpeechRecognition;

    if (SpeechRecognition) {
        recognition = new SpeechRecognition();
        recognition.continuous = true;       // Keep listening
        recognition.interimResults = false;  // Only final results
        recognition.lang = 'en-US';          // Language

        recognition.onresult = (event) => {
            const transcript = event.results[0][0].transcript;
            taskInput.value = transcript;

            if (event.results[0].isFinal) {
                stopRecording();
                if (transcript.trim()) {
                    sendText(transcript);  // Auto-send to avatar
                }
            }
        };

        recognition.onerror = (event) => {
            console.error('Speech recognition error:', event.error);
            showError('Speech recognition failed: ' + event.error);
            stopRecording();
        };
    }
}

    Configuration Options:

    Option Value Meaning
    continuous true Keeps listening until manually stopped
    interimResults false Only fires event for final transcription
    lang 'en-US' Recognition language

    Dark Mode Implementation

    JavaScript: 

    // Toggle theme on button click
themeToggle.addEventListener('click', () => {
    const theme = html.getAttribute('data-theme');
    const newTheme = theme === 'light' ? 'dark' : 'light';

    html.setAttribute('data-theme', newTheme);
    localStorage.setItem('theme', newTheme);  // Persist preference
});

    CSS Variables: 

    /* Light mode (default) */
:root {
    --bg-primary: #f5f5f5;
    --text-primary: #1a1a1a;
}

/* Dark mode */
[data-theme="dark"] {
    --bg-primary: #0f0f0f;
    --text-primary: #f0f0f0;
}

/* Usage */
body {
    background: var(--bg-primary);
    color: var(--text-primary);
}

    How it works:

    1. CSS defines variables for both light and dark themes
    2. JavaScript toggles data-theme attribute on <html>
    3. CSS selectors like [data-theme="dark"] override default variables
    4. localStorage remembers user preference across page loads
